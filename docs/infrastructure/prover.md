# Prover

The prover server generates Groth16 zero-knowledge proofs for Shade Protocol transactions. It receives a witness (circuit inputs) and returns a proof that the on-chain verifier contract can validate.

**Source:** [`packages/prover/`](https://github.com/shadeprotocolcom/shade-protocol/tree/main/packages/prover)

## API endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/prove` | Generate a Groth16 proof from a witness |
| `GET` | `/health` | Server health and prover type |

### POST /prove

Generate a proof from circuit inputs.

**Request body:** the witness JSON object containing all public and private signals for the JoinSplit circuit.

**Response:**

```json
{
  "proof": {
    "a": ["0x...", "0x..."],
    "b": [["0x...", "0x..."], ["0x...", "0x..."]],
    "c": ["0x...", "0x..."]
  },
  "publicSignals": ["0x...", "0x...", "0x...", "0x...", "0x...", "0x..."]
}
```

The proof is formatted for the on-chain verifier:
- `a` — G1 point (2 coordinates)
- `b` — G2 point (2x2 coordinates, swapped for EIP-197 compatibility)
- `c` — G1 point (2 coordinates)

### GET /health

Returns the server status and prover configuration.

**Response:**

```json
{
  "status": "ok",
  "proverType": "rapidsnark",
  "rapidsnarkPath": "/usr/local/bin/rapidsnark",
  "artifacts": {
    "wasmPath": "/app/artifacts/joinsplit.wasm",
    "wasmExists": true,
    "zkeyPath": "/app/artifacts/joinsplit.zkey",
    "zkeyExists": true
  }
}
```

## Prover backends

The server automatically detects the best available backend:

### rapidsnark (preferred)

[rapidsnark](https://github.com/nicklasso/rapidsnark) is a native binary prover that generates Groth16 proofs in under 1 second. The server searches for the binary in:

- `rapidsnark` (PATH)
- `/usr/local/bin/rapidsnark`
- `/usr/bin/rapidsnark`
- `${ARTIFACTS_DIR}/rapidsnark`

**Proof generation flow with rapidsnark:**

1. Calculate the witness using snarkjs WASM engine (`.wasm` file)
2. Write the witness to a temporary `.wtns` file
3. Run rapidsnark with the `.zkey` and `.wtns` files
4. Parse the output `proof.json` and `public.json`
5. Format for the on-chain verifier

### snarkjs (fallback)

If rapidsnark is not found, the server falls back to [snarkjs](https://github.com/iden3/snarkjs) WASM-based proving. This is significantly slower (15-45 seconds) but requires no native binary.

## Circuit artifacts

The prover requires two files in the artifacts directory:

| File | Description |
|---|---|
| `joinsplit.wasm` | WASM witness calculator (generated by circom) |
| `joinsplit.zkey` | Groth16 proving key (generated by snarkjs trusted setup) |

These files are produced by the circuit build process (`packages/circuits/scripts/build.sh`).

## Configuration

| Environment Variable | Default | Description |
|---|---|---|
| `PORT` | `5000` | HTTP server port |
| `ARTIFACTS_DIR` | `./artifacts` | Directory containing `.wasm` and `.zkey` files |

## Running locally

```bash
cd packages/prover

# Install dependencies
npm install

# Place circuit artifacts
cp ../circuits/artifacts/joinsplit.wasm artifacts/
cp ../circuits/artifacts/joinsplit.zkey artifacts/

# Run
npx tsx src/index.ts
```

## Docker

```bash
docker build -t shade-prover packages/prover/
docker run -p 5000:5000 \
  -v ./artifacts:/app/artifacts \
  shade-prover
```

## Self-hosting

Users who want zero-trust privacy can run their own prover. The only requirements are:

1. The circuit artifacts (`.wasm` + `.zkey`)
2. Node.js 20+ (or Docker)
3. Optional: rapidsnark binary for faster proving

Configure the SDK or frontend to point to `http://localhost:5000` instead of the public prover URL. When self-hosted, no private transaction data ever leaves the user's machine.
