# Architecture Overview

Shade Protocol is a clean-room implementation that follows RAILGUN's proven cryptographic architecture, optimized for Citrea's Bitcoin L2 environment.

## Design principles

1. **Follow RAILGUN's proven patterns** — the cryptographic design is battle-tested with real money at scale
2. **Keep it minimal** — only features needed for the MVP, no premature abstraction
3. **Clean-room code** — inspired by RAILGUN's architecture, not copying their UNLICENSED code
4. **Citrea-optimized** — cBTC native asset, 2-second blocks, 10M gas block limit

## Simplifications vs. RAILGUN

Shade Protocol makes deliberate simplifications for the MVP phase:

| Feature | RAILGUN | Shade (MVP) |
|---|---|---|
| Token support | ERC-20 + ERC-721 | Only cBTC (native + WcBTC, auto-wrapping) |
| Relay / Adapt | Complex relay contract | User pays gas directly |
| Fee model | Shield + unshield fees | No fees initially |
| Broadcaster | Waku network | User submits directly (see [privacy risks](/risks#privacy-model)) |
| PPOI | Full compliance system | Deferred to later phase |
| Multiple trees | Auto-create on overflow | Single tree, 65K capacity (see [tree capacity](/risks#tree-capacity)) |
| Circuit variants | 54 circuits (1x1, 1x2, 2x2, etc.) | 2x2 only (dummy inputs for single-note txs) |

## Proof system

Shade Protocol uses **Groth16 over BN254** for zero-knowledge proofs. The verifier contract is generated directly by `snarkjs groth16 exportsolidityverifier`, which produces a self-contained Solidity file with pairing operations, the verifying key, and the `verifyProof()` function.

Proof generation happens server-side using [rapidsnark](https://github.com/nicklasso/rapidsnark), completing in under one second. A [snarkjs](https://github.com/iden3/snarkjs) WASM fallback is available when rapidsnark is not installed.
